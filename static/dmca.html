DMCA & Content Policy

Last Updated: October 17th, 2025

Understanding Nymchat Architecture

Nymchat is a decentralized, pseudonymous messaging application that operates differently from traditional chat services. It's important to understand how content is distributed on our platform:

  • Nymchat does not host any message or media content. We are a client/interface that displays content stored elsewhere.
  • All media files (images, videos) are stored on independent Blossom servers operated by third parties.
  • Message content and metadata are published to Nostr relays, which are also independently operated.
  • Nymchat simply aggregates and displays this distributed content through a user-friendly interface.

Filing a DMCA Claim

Because Nymchat does not host content, we cannot directly remove infringing material. To address copyright infringement, you must contact the appropriate parties:

1. For Image/Video Content

Media files are stored on Blossom servers. You need to:

  • Identify which Blossom server is hosting the content (the URL will indicate this)
  • Contact the Blossom server operator directly with your DMCA takedown request
  • Each Blossom server operates independently and has its own policies

2. For Messages/Metadata

If the infringing content is in a message, profile, channel name, or other metadata:

  • This information is stored on Nostr relays
  • Contact the specific Nostr relay operators where the content appears
  • Multiple relays may store the same data due to Nostr's distributed nature

What Nymchat Can Do

While we don't host content, we can:

  • Help you identify which Blossom servers or relays are hosting specific content
  • Add content to a local blocklist so it doesn't appear in our interface (note: this only affects nymchat.app, not other Nostr clients)
  • Provide guidance on how to contact the appropriate service operators

To request assistance, please contact us with:

  • The URL or identifier of the infringing content
  • Proof of your copyright ownership
  • A description of the copyrighted work

Blossom Server Operators

If you operate a Blossom server and receive a DMCA takedown request, you are responsible for handling it according to your jurisdiction's laws.

For a list of known Blossom servers, refer to the Blossom project documentation.

Important Considerations

  • Decentralization: Due to the decentralized nature of Nostr and Blossom, complete removal of content from all sources may not be possible.
  • Multiple Copies: Content may exist on multiple Blossom servers and relays. You may need to contact multiple operators.
  • Hash-Based Storage: Blossom uses content-addressed storage. The same file will have the same hash on any server.
  • No Central Control: Neither Nymchat nor any single entity has the ability to remove content from the entire network.

Counter-Notification

If you believe your content was wrongly removed from a Blossom server, you should contact that server operator directly to file a counter-notification.

Contact
